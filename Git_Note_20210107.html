安裝

  • 在 macOS 有 Homebrew 非常方便
    • brew install git
  • 安裝之後，無論是在 Win 或 macOS，在應用程式 Git Bash 中，
    都可以進入像是 Linux 一樣的 Bash 命令列程式
  • which git
    • 檢視 git 在哪裡？
    • 原本執行的 Git 的位置是在 /usr/bin/git/
    • 使用 Homebrew 安裝的 Git 執行的
      位置改成了 /usr/local/bin/git/
      這是 Homebrew 安裝所有套件預設的位置。
  • git --version
    • 檢視使用版本

使用者設定

  • git config --list
    • 檢視目前的設定
  • git config --global user.name "ChiaYao Chen"
    • 第一次執行 git 請設定使用者名稱
  • git config --global user.email "chiayao.chen@gmail.com"
    • 第一次執行 git 請設定使用者信箱
  • 直接編輯 ~/.gitconfig 這個文字檔案也可以
  • 如果要在不同專案設定不同使用者與信箱，則把　--global 換成　--local 就可以了，例如
    • git config --local user.name "ChiaYao Chen"
    • git config --local user.email "chiayao.chen@gmail.com"

開始使用

  • git init
    • 初始化當下的工作目錄，讓 git 對當下的工作目錄進行版本控制
    • 執行完之後就會建立一個 .git 目錄
    • 如果要解除版本控制，可以直接刪掉 .git 目錄
  • git status
    • 查看目前工作目錄的 git 狀態
  • git add --all
    • 把所有改變加入暫存區 Staging area，包括檔案的刪除等
  • git commit -m "在這次的提交做了什麼事情"
    • 執行後就會把這次 Staging area 有記錄到的改變，放到 Repository 裡面
    • 請注意 建立新的空目錄是無法被提交的，如果有需求，則可以故意在新的空目錄裡面隨意放一個空的檔案，例如
      • touch 空目錄/.gitKeepThisDir
  • git log
    • 查看 Commit 的歷史記錄

救回檔案

  • 如果誤刪檔案，但還沒提交到暫存區
    • git checkout 檔案名稱
    • git checkout .
      • 檔案名稱改為句點，則代表所有檔案
  • 如果已經 commit 了
    • git reset HEAD^
      • 每一個 ^ 符號代表回退一步
      • 這代表把 HEAD 退回前一個 commit
      • 如果是 HEAD^^ 則是退兩步

使用分支

  • git branch
    • 顯示有哪些分支，以及目前位置
  • git branch 分支名稱
    • 新增分支，但這樣不會移動到新增的分支上
  • git branch -m 舊的分支名稱 新的分支名稱
    • 更改分支名稱
  • git branch -d 想刪除的分支名稱
    • 前提是要 fully merged 已完全合併的分支才可以刪
    • 如果參數改成 -D 則不管，可以強制刪除
  • git checkout 想切換過去的分支名稱
    • 切換所在的分支位置
    • 前提是分支必須先存在
  • git checkout -b 新的分支名稱
    • 建立新的分支並且切換過去
  • 如果還沒 add 或 commit 就切換到其他分支
    • 不要害怕，剛才所做的努力都還在，只有commit時會到不同地方
  • git merge 想要合併過來的分支名稱
    • 已經合併過的分支，如果再也不要，也可以刪了
    • 如果合併發生衝突 conflict 可以使用
      • git checkout --ours 衝突的檔案名稱
      • git checkout --theirs 衝突的檔案名稱
      • 來決定用我的檔案還是他的檔案

遠端協作

  • git remote add origin git@github.com:倉庫位置
    • 代表增加一個遠端節點
    • origin 是後面遠端節點的代號，可以自己取其他名字
  • git push -u origin master
    • 把我現在的 master 內容，推到 origin
    • 有加 -u 參數表示設定 upstream 這樣之後
      偷懶打 git push 不加其他參數 也會進行一樣的動作
  • git fetch
    • 從前面設定的遠端節點抓更新下來，並且放到新的分支裡
    • 通常新的分支名稱會叫做 origin/master
    • 後續想要合併可以打 git merge origin/master
      才會跟現在的分支合併
  • git pull
    • 直接自動做完 git fetch 與 git merge 的動作
    • 相當於從線上更新目前的分支
  • git clone git@github.com:倉庫位置
    • 看到有趣的專案，這個指令可以
      把整個專案複製下來並且存在同名的目錄裡面
    • 若 clone 完成，之後想要更新，
      則使用 git pull 即可

使用GitHub放靜態網頁

  • 基本說明在 https://pages.github.com/
  • HelloWorld
    • 創造一個專案名稱叫做 username.github.io
    • git clone https://github.com/username/username.github.io
    • 進入自己電腦的專案資料夾創建 index.html
    • 其他一樣
      • git add --all
      • git commit -m "Initial commit"
      • git push -u origin main
    • 然後網址 username.github.io 就會開啟 index.html
  • 如果想要獨立專案有自己的倉庫
    • 先新增一個 repo 假設名字叫 projectX
    • 把 projectX 整個 git clone 到自己電腦資料夾
    • 建立 index.html
    • 一樣先 git add 與 git commit
    • 接著是重點
    • git checkout -b gh-pages
    • 然後把 gh-pages 推回去
    • git push -u origin gh-pages
    • 那麼在網址 username.github.io/projectX/
    • 就是屬於這個倉庫的網頁了

SSH的問題造成無法上傳檔案

  • 在使用者的主目錄可以檢查看看
    • 有沒有 .ssh 目錄
    • 有沒有 id_rsa 這個檔案
    • 有沒有 id_rsa.pub 這個檔案
  • 如果有缺，則必須自己建立新的
    ssh-keygen -t rsa -C "chiayao.chen@gmail.com"
    • 參數 -t 用以指定生成密鑰的類型
    • 參數 -C 用以註記
    • 參數 -f 用以指定生成密鑰的檔案名稱，預設是 id_rsa
      私鑰為 id_rsa，公鑰為 id_rsa.pub
    • 其餘的設定過程都可以直接 enter 跳過
  • 完成之後在 /Users/chiayao/.ssh 目錄下面應該可以找到
    公鑰 id_rsa.pub
  • 在網頁 https://github.com/settings/keys 裡面
    • 按下 New SSH key
    • open id_rsa.pub -e
      把公鑰 id_rsa.pub 的內容複製貼上
    • 成功增加了這台電腦的 SSH key 了
